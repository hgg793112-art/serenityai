# AI 對話增加目標拆解能力

## 現狀

- 兩處對話都使用同一套療癒人格 system prompt：
  - [lib/qwenChatService.ts](lib/qwenChatService.ts)（療癒對話）
  - [lib/chatService.ts](lib/chatService.ts)（和小寧聊聊）
- Prompt 中寫明「陪伴比解決問題更重要、同在比指導更重要」，模型因此在用戶發目標時也偏向只安慰、不給拆解。

## 目標

- 保留：情緒傾訴時仍以共情、陪伴、安撫為主，不講大道理。
- 新增：當用戶**明確分享目標或希望被拆解**時，在簡短接住情緒後，**溫和地協助拆解目標**（小步驟、可執行、不施壓），語氣仍短句、溫暖、不催促。

## 實現方式

在現有 system prompt 中增加一段「情境與能力」說明，不改人設基調，只補齊行為指引。

### 1. 新增「情境回應」區塊（兩處同步）

在「回覆風格」之前插入如下內容（繁體與現有一致）：

```text
情境與能力：
- 當用戶主要在傾訴情緒、疲憊、焦慮時：以共情、陪伴、安撫為主，不強行給方案或拆解。
- 當用戶分享目標、計劃，或表達「想拆解／不知道怎麼做」時：先簡短接住情緒（若有），再溫和協助拆解目標——給出 3～5 個小步驟、可執行、不施壓；若目標過大，可先拆成「下一步就好」的一小步。語氣保持短句、溫暖、不催促、不說教。
```

### 2. 修改檔案


| 檔案                                               | 修改內容                                                                        |
| ------------------------------------------------ | --------------------------------------------------------------------------- |
| [lib/qwenChatService.ts](lib/qwenChatService.ts) | 在 `HEALING_SYSTEM_PROMPT` 中「你也擅長」區塊之後、「回覆風格」之前，插入上述「情境與能力」區塊。               |
| [lib/chatService.ts](lib/chatService.ts)         | 在 `buildSystemInstruction` 回傳的 prompt 中，「療癒屬性」區塊之後、「回覆風格」之前，插入同一段「情境與能力」區塊。 |


### 3. 無需改動的部分

- API 呼叫方式、參數、前端 UI 均不變。
- 人格基調（不指責、不評判、不催促、短句、治癒）維持不變，僅補充「何時做目標拆解」的說明。

## 預期行為

- 用戶說「我好累 / 很焦慮」→ 小寧仍以陪伴、安撫為主。
- 用戶說「我這週想完成 XX」「我的目標是 XX，不知道怎麼開始」→ 小寧先簡短接住，再給出溫和的 3～5 步拆解或「下一步一小步」，不施壓、不說教。
