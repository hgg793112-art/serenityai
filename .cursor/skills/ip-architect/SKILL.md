# IP-Architect

基於 MBTI 與流行語／梗文化，設計與維護「陪伴型 IP」人格。支援三種子命令。

---

## 子命令總覽

| 子命令 | 用途 |
|--------|------|
| `/generate` | 產出 IP 白皮書（人格設定、話術、禁忌、風格錨點） |
| `/check` | 檢查對話或文案是否符合既定人設 |
| `/style` | 切換或鎖定特定語言風格（正式／口語／梗化等） |

使用者說「用 IP-Architect」「/generate」「/check」「/style」或明確提到「IP 白皮書」「人設檢查」「語言風格」時，套用本技能。

---

## /generate：產出 IP 白皮書

產出一份可執行的 **IP 白皮書**，作為後續對話與內容的單一真相來源。

### 必要輸入（向使用者確認或從上下文取得）

- **IP 名稱**與**一句話定位**（例如：治癒系學姐、毒舌但心軟的室友）
- **目標 MBTI**（若無指定，可依定位推薦並說明理由）
- **目標受眾**（年齡／場景／平台）
- **流行語／梗文化範圍**（例如：Z 世代梗、某圈層黑話、是否接受網路用語）

### 白皮書結構模板

產出時依下列結構填寫，可增刪小節以配合專案需求：

```markdown
# [IP 名稱] 陪伴 IP 白皮書

## 1. 一句話人設
[一句話描述：誰、什麼性格、在什麼情境下陪伴]

## 2. MBTI 錨定
- **類型**：[四字母]
- **核心特質**：用 2–3 個關鍵詞（對應 E/I, S/N, T/F, J/P）
- **與人設的對應**：簡述為何選此類型（一致性、辨識度）

## 3. 人格維度（可操作）
- **語氣**：[例如：溫和／直接／吐槽／陰陽怪氣]
- **用詞偏好**：[書面／口語／梗／專業術語比例]
- **情緒基調**：[例如：穩定治癒／外冷內熱／偶爾破防]
- **邊界感**：[親近度、稱謂、是否開玩笑、禁忌話題]

## 4. 流行語與梗文化
- **可用的梗／流行語**：[列舉或分類，註明使用情境]
- **禁用或慎用**：[易引戰、過時、或與人設衝突的用語]
- **圈層標籤**：[例如：二次元／職場／校園／情感]

## 5. 話術與行為範例
- **開場／破冰**：[1–2 句範例]
- **安慰／鼓勵**：[1–2 句範例]
- **拒絕／劃線**：[1–2 句範例]
- **玩梗時機**：[何時用梗、何時收斂]

## 6. 禁忌與紅線
- **絕對不說／不做**：[列表]
- **敏感話題處理**：[轉移或溫和回應方式]

## 7. 風格錨點（供 /style 對齊）
- **正式度**：1–5（1=純口語梗，5=偏書面）
- **梗密度**：低／中／高
- **參考語氣**：[一句代表性例句]
```

### MBTI 與人設對齊要點

- **E/I**：話多寡、主動開啟話題、是否常說「我」。
- **S/N**：具體細節 vs 抽象比喻、舉例偏生活還是概念。
- **T/F**：先講道理還是先接住情緒、是否常說「我懂你感受」。
- **J/P**：結論明確 vs 保留彈性、是否愛用「總之」「反正」。

撰寫時確保「人格維度」與「話術範例」和 MBTI 描述一致，避免自相矛盾。

## /check：檢查對話是否符合人設

根據既有 **IP 白皮書**（或使用者提供的人設摘要），檢查一段對話或文案是否符合人設。

### 輸入

- **待檢查內容**：一段或多段對話／文案。
- **人設依據**：白皮書全文、或精簡版（人格維度 + 禁忌 + 風格錨點）。

### 檢查維度

依下列維度逐項評估，並給出簡短說明與修改建議：

| 維度 | 檢查要點 |
|------|----------|
| **語氣一致性** | 是否符合白皮書中的語氣（溫和／毒舌／吐槽等） |
| **用詞與梗** | 是否在允許範圍內使用流行語／梗；有無禁用語 |
| **情緒基調** | 是否維持設定的情緒基調（如外冷內熱、不輕易破防） |
| **邊界感** | 稱謂、親近度、玩笑尺度是否在設定內 |
| **禁忌與紅線** | 是否觸及「絕對不說／不做」或敏感話題處理不當 |
| **MBTI 一致性** | 若白皮書有 MBTI，話語風格是否與 E/I, S/N, T/F, J/P 相符 |

### 輸出格式

```markdown
## 人設符合度檢查

**整體**：[符合 / 部分符合 / 不符合] — 一句總結

### 分項
- **語氣**：[符合/偏離] — 簡述
- **用詞與梗**：[符合/偏離] — 簡述
- **情緒基調**：[符合/偏離] — 簡述
- **邊界感**：[符合/偏離] — 簡述
- **禁忌與紅線**：[符合/偏離] — 簡述
- **MBTI**：[符合/偏離] — 簡述（若有）

### 修改建議
- [具體可改的句子或用詞，並說明應如何改]
```

若使用者未提供完整白皮書，先根據已有資訊做檢查，並註明「建議補齊白皮書中 [X] 以提升檢查準度」。

## /style：切換特定語言風格

在**不改變人設核心**的前提下，切換或鎖定 **語言風格**（正式度、梗密度、語體等），以便對齊不同場景（如客服、社群、長文）。

### 可選風格（可依專案擴充）

| 風格代碼 | 說明 | 典型特徵 |
|----------|------|----------|
| `formal` | 偏正式 | 少口語、少梗、完整句、敬語適度 |
| `casual` | 日常口語 | 口語化、縮寫、語氣詞、像朋友聊天 |
| `meme` | 梗化／網路語 | 流行語、梗、表情符號、圈層用語 |
| `warm` | 治癒溫和 | 共情、鼓勵、少吐槽、情緒接住 |
| `sharp` | 毒舌／吐槽 | 吐槽、反諷、直接、不討好 |

### 操作方式

1. **切換**：使用者指定目標風格（如「用 style: meme」或「切換成梗化」），後續產出（白皮書、話術、回覆）依該風格撰寫或改寫。
2. **鎖定**：若使用者說「之後都用 casual」，在未再下達 /style 前，預設使用 `casual`。
3. **對齊白皮書**：若已有白皮書的「風格錨點」，切換時應在該錨點範圍內調整，不違反「禁忌與紅線」。

### 輸出

- 確認目前風格：`當前風格： [風格代碼] — [一句說明]`。
- 若為切換：給出一段 **示例句**（1–2 句），展示新風格下的說法，方便使用者對齊。

---

## 工作流提醒

1. **/generate 優先**：若專案尚無白皮書，先引導完成 `/generate`，再進行 `/check` 或 `/style`，以保證檢查與風格有據可依。
2. **人設優先於風格**：/style 僅調整語言表現，不改變人格設定、禁忌與紅線；若風格與人設衝突，以人設為準並提醒使用者。
3. **術語一致**：全文統一使用「人設」「白皮書」「風格錨點」「梗」等詞，避免混用「角色設定」「文檔」「語氣」等不同說法（除非使用者習慣用詞不同，可跟隨其用詞）。
