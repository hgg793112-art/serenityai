# 小寧陪你 — 操作步驟

本地運行與語音功能所需步驟，按順序執行即可。

**網頁操作**：若你希望由 Agent 代為打開網頁並操作，可直接說「用 MCP 幫我取得 Gemini Key」或「用 MCP 幫我設定 Supabase」。需要登入的頁面會請你在瀏覽器裡登入，完成後回覆，Agent 會再取得所需資訊並寫入設定、繼續後續步驟。

---

## 一、本地運行（必做）

### 步驟 1：安裝依賴

在專案根目錄執行：

```bash
npm install
```

### 步驟 2：設定 Gemini API Key（語音／麥克風必填）

1. 打開 [Google AI Studio](https://aistudio.google.com/apikey)，登入後點 **Create API Key**。
2. 複製產生的 API Key。
3. 在專案根目錄打開或新建 **`.env.local`**。
4. 加入或修改這一行（把 `你的_Gemini_API_Key` 換成剛複製的金鑰）：

   ```
   VITE_GEMINI_API_KEY=你的_Gemini_API_Key
   ```

5. 存檔。

### 步驟 3：啟動開發伺服器

在專案根目錄執行：

```bash
npm run dev
```

終端出現 `Local: http://localhost:3000/` 後，用瀏覽器打開：

**http://localhost:3000**

即可預覽 App；進入「壓力」頁並點開始，即可使用語音對話（需允許麥克風權限）。  
在首頁點「小寧聊天」→「和小寧聊聊（有記憶的對話）」即可使用**有近期對話記憶**的文字對話（需已設定 `VITE_GEMINI_API_KEY`）。

---

## 二、對話記憶表（Supabase，選做）

若已使用 Supabase 且希望**對話記錄存到雲端**，在 Supabase 專案恢復/可用後：

1. 打開 [Supabase Dashboard](https://supabase.com/dashboard) → 選擇專案 → 左側 **SQL Editor** → **New query**。
2. 貼上專案內 **`supabase/migration_chat_memory.sql`** 的完整內容，點 **Run**。
3. 執行成功後，`chat_messages` 與 `memory_facts` 表即建立；未執行時 App 會自動用 localStorage 存對話。

---

## 三、使用 Supabase 存心情日誌（選做）

若希望心情日誌存到雲端、多裝置同步，再執行以下步驟。

### 步驟 4：建立 Supabase 專案並取得金鑰

1. 打開 [Supabase 儀表板](https://supabase.com/dashboard)，建立或選擇一個專案。
2. 左側 **Project Settings** → **API**。
3. 複製 **Project URL** 與 **anon public** key。

### 步驟 5：寫入 .env.local

在專案根目錄的 **`.env.local`** 中新增或修改：

```
VITE_SUPABASE_URL=你的_Project_URL
VITE_SUPABASE_ANON_KEY=你的_anon_公鑰
```

存檔。

### 步驟 6：建立資料表

1. 在 Supabase 左側點 **SQL Editor** → **New query**。
2. 打開專案裡的 **`supabase/schema.sql`**，複製全部內容貼到 SQL 編輯器。
3. 點 **Run** 執行。

### 步驟 7：重啟開發伺服器

若 `npm run dev` 已在運行，按 `Ctrl+C` 停止後再執行一次：

```bash
npm run dev
```

之後心情日誌會存到 Supabase；未設定上述變數時仍使用瀏覽器 localStorage。

---

## 四、打包 Android APK（選做）

在專案根目錄執行：

```bash
npm run build
npx cap sync android
```

再用 Android Studio 打開 **`android`** 資料夾，或使用專案內建腳本：

```bash
npm run build:android
```

（需已安裝 Android SDK 與相關環境。）
